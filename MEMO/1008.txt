PK(unique,not null) : 기본키 (주민등록번호 같은거) --- 무조건 있어줘야함.
CHAR vs VARCHAR ( CHAR는 고정값 메모리. VARCHAR는 유동적인 메모리.)
DATETIME : 날짜와 시간 저장
.
NULL : NULL을 허용한다
NOT NULL : NULL을 허용하지 않는다.
------------------------------------------------------------------------------------------------------
unsigned : 양의 정수로만 취급. (-10~10 ---> 0~20)
auto_increment : PK값 자동으로 올려줌. --- 편안.

check : 제약 조건. ('남','여') 이 중 하나만 가능. 
[예시 : gender CHAR(1) NOT NULL CHECK(gender IN('남','여'))]

트랜잭션 commit, rollback, 은행 일련의 업무가 예시.(ACID)

복습정리

SELECT (보고 싶은 컬럼명 or 컬럼명들 / * ) FROM 테이블명   
WHERE (조건식) 

GROUP BY(컬럼명 or 컬럼명들)   
HAVING (조건식)

ORDER BY (컬럼명 or 컬럼명들)

LIMIT 10,10 >> 인덱스값, 갯수   >>  10번 인덱스값에서 10개 보여짐. 
LIMIT 10  >>0번 인덱스부터 10개.

INSERT INTO (테이블명) (컬럼명 or 컬럼명들) VALUES (값 or 값들)

DELETE FROM (테이블명) WHERE (조건식) 	델프웨

UPDATE (테이블명) SET (컬럼명)=(수정할값),(컬럼명)=(수정할 값) WHERE (조건식)   업셋웨
